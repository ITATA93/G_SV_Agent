## [Unreleased]

### Added
- **`configs/services.yml`** — Single Source of Truth for all 24 infrastructure services (replaces hardcoded lists)
- **`scripts/sync_service_catalog.py`** — Auto-generates `verify-services.sh` and `docs/SERVICE_CATALOG.md` from services.yml
- **`docs/SERVICE_CATALOG.md`** — Auto-generated catalog with all services, ports, versions, and NocoBase detail section
- **MIRA/MIRA2 NocoBase** tracked in `.env`, `services.yml`, health checks (were deployed 2026-02-26 but undocumented)
- **ClickHouse** added to service registry (was in deploy script but missing from health checks)
- **Service Registration rule** in `.agent/rules/project-rules.md` — mandatory procedure for service changes
- **Infrastructure sync check** in session-protocol, update-docs, project-status, auto-memory
- **`configs/consumers.yml`** — Maps satellite projects (G_Dashboard, G_Plantilla, G_Orion, GEN_OS-master) to the credentials they consume from G_SV_Agent
- **`--propagate` flag** in `sync_service_catalog.py` — reads `consumers.yml` and updates satellite `.env` files with master credentials
- **`--propagate-only` flag** — propagate credentials without regenerating catalog
- **Transform support** in credential propagation: `external_url` resolves service external URLs from `services.yml`, `internal_url` uses master .env values as-is
- **Credential Propagation rule** in CLAUDE.md, GEMINI.md, project-rules.md, deployer.md, session-protocol.md

### Changed
- `src/health.py` — loads services from `configs/services.yml` instead of hardcoded list (24 services, was 22)
- `scripts/verify-services.sh` — now auto-generated by sync script (23 external checks + ClickHouse internal-only)
- `.env` — added MIRA (port 13003, v1.9.14) and MIRA2 (port 13002, v2.0.6) NocoBase sections
- `.env.example` — reorganized into Local Dev / Production Infrastructure / External Integrations sections
- `docs/ACTIVATION_CHECKLIST.md` — corrected status: NocoBase/MIRA/MIRA2 marked as deployed (were incorrectly shown as DOWN)
- `docs/OPERATIONS.md` — rewritten from 22-line stub to comprehensive production operations guide (150+ lines)
- `docs/library/scripts.md` — registered `sync_service_catalog.py`, updated verify-services.sh description

### Fixed
- Documentation no longer contradicts reality (ACTIVATION_CHECKLIST said "NocoBase DOWN" when DEVLOG said deployed)
- Service count corrected: 24 total (was claiming 22, missing MIRA2 and ClickHouse)
- `.env` header date updated from 2026-02-05 to 2026-03-02

- **`src/` server agent module** -- Python package for infrastructure management
  - `src/__init__.py` -- Package init with version, exports
  - `src/agent.py` -- `ServerAgent` class: `check_services()`, `deploy_service()`, `restart_service()`, `start_service()`, `stop_service()`, `get_logs()`, `list_containers()`, `container_status()`, `test_connectivity()`, `disk_usage()`
  - `src/health.py` -- `HealthChecker` with all 22 services, external HTTPS and internal SSH checks, `HealthReport` with JSON export
  - `src/config.py` -- `Config` dataclass loader from `.env` with typed sub-configs (`PostgresConfig`, `SSHConfig`, `LangfuseConfig`)
- **`scripts/fix_prometheus.sh`** -- Restart Prometheus container on VM101 via SSH and verify health
- **`scripts/init_portainer.sh`** -- Reset and initialize Portainer admin account on VM101 via API
- **`docs/CREDENTIAL_SETUP.md`** -- Step-by-step guide for obtaining all 6 external credentials: Langfuse, Notion, Azure AD, Gmail OAuth, Tailscale, Docker Desktop
- **`.env.example` expanded** -- Added 15 new placeholder variables: Langfuse, Notion, Azure AD, Gmail OAuth, Tailscale, Portainer, Prometheus, PostgREST JWT, VM101 SSH config
- **Autopilot mode**: Fully autonomous execution mode (zero user intervention after mission brief)

  - `.agent/rules/autopilot-guardrails.md` — Automatic safety: branch isolation, scope enforcement, kill switches

  - `.claude/commands/autopilot.md` — `/autopilot` slash command for Claude Code

- Post-flight reporting: automatic summary of all changes, decisions, and test results
- Governance audit: docs/TODO.md created
